# QQ 机器人发布（上线）操作指引

## 关键认知
- 你注册的机器人当前状态：**开发中**
- 主动消息 API 权限：**仅在"已上线"后开通**
- **不需要创建任何 QQ 频道**，只用 QQ 群聊场景
- 发布审核时间：**1-3 个工作日**

## ⚠️ 重要限制（必须先知道）
- **群聊主动消息每月 4 条**（平台硬限制，企业/个人认证都不能突破）
- **单聊主动消息每月 4 条**（C2C 互动召回）
- **被动回复无限制**（5 分钟窗口内，可回复 5 次）

**为什么 4 条不够用**：
你满仓被套，盘中可能有多个信号触发。按月 4 条根本不够。

**应对方案**：
1. **主用被动回复**（你 @ 机器人查信号 → 机器人立刻返回）—— **无限次**
2. **强信号（每月 4 条额度）** 用于关键预警（跌破止损、紧急买卖点）
3. **搭配微信通道**（wechatbot-webhook）做冗余 — 没有月度限制

## 操作路径（逐步）

### 1. 进入机器人管理端
1. 访问 https://q.qq.com
2. 用你的 QQ 扫码登录
3. 顶部「我的机器人」/「机器人列表」→ 找到 1903580590
4. 进入该机器人的管理端

### 2. 完成基础信息设置
左侧菜单 → **「基础信息」**
- 机器人昵称：随便起（不能用"QQ""腾讯"等官方词，不能含广告词）
- 机器人头像：上传图片
- 机器人简介：描述这个机器人的功能（一两句话）
- 隐私协议 URL：随便填个 https://example.com 就行
- 开发者信息：填你的联系方式（邮箱）

### 3. 开发场景选择（关键）
左侧菜单 → **「开发设置」** → **「开发场景」**
- ✅ 勾选「**QQ 群聊**」
- ✅ 勾选「**消息列表单聊**」（私聊推送）
- ❌ **不需要勾选**「QQ 频道」

### 4. 开发基础设置
左侧菜单 → **「开发设置」** → **「基础设置」**
- 填一个**回调地址**（随便填个 https://example.com 就行）

### 5. 指令配置（可跳过）
这一步先跳过，不影响主动消息能力。后续做命令交互时再回来配。

### 6. 使用范围配置
左侧菜单 → **「发布设置」** → **「使用范围」**
- 选 **「全量」**（最简单）
- 或选「白名单」→ 加你的测试群 group_openid: `9AF12D111FCCA67624811005E51EC4C2`

### 7. 自测报告（关键步骤）
左侧菜单 → **「发布设置」** → **「自测报告」**
- **下载自测报告模板**（页面里）
- 模板会列出需要测试的功能清单
- 按模板测试，**填上测试结果截图或说明**
- 上传
- 这步最耗时（30-60 分钟）

### 8. 提交审核
左侧菜单 → **「发布」** / **「提交审核」**
- 检查所有必填项是否完成
- 点「提交审核」
- 等待 1-3 个工作日

### 9. 审核通过 → 手动上线
- 审核通过后会收到通知
- 进入机器人管理端
- 找到 **「上线」** 按钮
- **手动点"上线"**

### 10. 验证主动消息权限
机器人状态变成 **「已上线」** 后，跑测试：
```bash
cd /Users/jojo/code/a-trade
./start.sh scripts/test_openclaw.py
```
成功的话会看到群里收到测试消息。

2. 我开始写被动回复 + 命令路由（你 @ 机器人就能响应，无月度限制）
3. 审核通过后切换到主动推送（每月 4 条额度用于强信号）

回我 "开始被动回复" 我立刻开始写。

## 审核通过后的真实使用模式

由于主动消息每月只有 4 条，建议**主用被动回复 + 4 条主动额度留给强信号**：

| 场景 | 推送方式 | 月度限制 |
|---|---|---|
| 你想看持仓股当前所有信号 | `@机器人 /signal`（被动回复） | 无限制 |
| 你想看个股详情 | `@机器人 /check 600519`（被动回复） | 无限制 |
| 你想看收盘报告 | `@机器人 /report`（被动回复） | 无限制 |
| 跌破止损 / 紧急买点 | a-trade 主动推送（用月度 4 条额度） | 4 条/月 |
| 普通盘中信号 | 不主动推，你 @ 机器人查 | 无限制 |

这样**既符合 QQ 平台规则，又能满足你"手机能用"+"主动获取信号"的所有需求**。
