# QQ 机器人主动消息权限开通指引

## 现状
你的机器人目前状态：**开发中**
错误码 `40034105 "主动消息失败, 无权限"` — API 调用被平台拒绝
根本原因：**"开发中"状态未开通"主动推送" API 权限**

## 必须完成的步骤（按顺序）

### 第 1 步：登录开发者后台
访问：https://q.qq.com
用创建机器人的 QQ 号扫码登录

### 第 2 步：完成主体认证（如果还没做）
路径：右上角头像 → 账号管理 / 主体信息
- **个人认证**：填身份证 + 手机号（必须与运营商实名一致）
- **企业认证**：营业执照 + 对公账号 + 打款验证（更复杂，不推荐个人用户）

⚠️ **关键**：未认证 = 30 qpm；个人认证 = 60 qpm
**当前报错很可能就是未认证主体被卡了**

### 第 3 步：在机器人管理端完成基础信息
路径：q.qq.com → 我的机器人 → 选中你的机器人（1903580590）→ 左侧菜单「基础信息」

填写：
- 机器人昵称
- 机器人头像
- 机器人简介
- 开发者信息

### 第 4 步：配置开发场景和权限（关键）
路径：机器人管理端 → 「开发设置」/「能力配置」

**重点检查 4 个开关**：

1. **开发场景选择**
   - 勾选 "**QQ 群聊**"（主动消息的关键场景）
   - 勾选 "**消息列表单聊**"（私聊推送）
   - 勾选 "**QQ 频道**"（可选）

2. **消息能力**
   - 找到 "**主动推送群消息**" 或 "**send_group_msg 主动能力**"
   - **打开**

3. **事件订阅 Intents**
   - 至少勾选 `GROUP_AT_MESSAGE_CREATE`（接收群 @ 消息）
   - `C2C_MESSAGE_CREATE`（接收私聊消息）
   - `GROUP_MESSAGE_CREATE`（接收普通群消息）

4. **使用范围**（必须）
   - 选 "**全量**" 或 "**白名单**"（白名单需要把测试群加进去）
   - "开发中" 状态下建议先选 **全量**（方便测试）

### 第 5 步：完成机器人发布流程
路径：机器人管理端 → 「发布」/「上线」

按提示完成：
1. **机器人资料审核**（提交机器人简介、头像、隐私协议）
2. **能力清单自测**（按模板测试功能，下载自测报告填写后上传）
3. **提交发布审核**（腾讯审核，一般 1-3 个工作日）
4. **手动上线**（审核通过后点 "上线" 按钮）

⚠️ **关键提示**：
- **审核通过并手动上线前**，主动消息权限不会真正生效
- 上线后机器人会被 QQ 搜索到（私密项目慎用）

### 第 6 步：上线后再回来测试
机器人状态变成 "**已发布/已上线**" 后，再跑：

```bash
cd /Users/jojo/code/a-trade
./start.sh scripts/test_openclaw.py
```

如果还报 `40034105`，等 5-10 分钟（权限同步可能有延迟），再试。

## 速查表：常见错误码

| 错误码 | 含义 | 解决 |
|---|---|---|
| `40034105` 主动消息失败, 无权限 | 机器人未发布/未开通主动能力 | 完成第 4、5 步 |
| `1100301` 调用方不是群成员 | Bot 未在该群 | 在群里添加机器人 |
| `1100304` @次数鉴权失败 | 调用过于频繁 | 降低推送频率 |
| `22009` msg limit exceed | 触发月/日配额 | 等待下个周期或申请提额 |
| `40011000` 请求数据异常 | payload 格式错 | 检查 msg_seq、msg_type 是否为整数 |

## 我的判断（基于已有信息）

你之前说"解锁了全部权限"但 API 还报 `40034105`，最可能的原因是：

1. **机器人状态没变** — 后台显示还是"开发中"，权限没有真正生效
2. **发布审核没完成** — 个人开发者必须完成发布审核才能用主动 API
3. **认证状态问题** — 未认证主体的主动消息能力可能仍受限

**最快路径**：完成第 2 步（个人认证） + 第 5 步（提交发布审核），等审核通过后状态自动变成"已上线"，主动消息 API 就能用了。
