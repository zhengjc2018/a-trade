# 项目协作规范

## 任务进度追踪

对于需要修改代码、跨多个步骤或进行系统性分析的任务，开始执行前必须：

1. 在 `docs/progress/YYYY-MM-DD-<task-name>/` 创建 `TODO.md`。
2. 在同一目录创建 `STATUS.md`。
3. `TODO.md` 使用复选框列出可验证步骤，并维护未开始、进行中、已完成、受阻状态。
4. `STATUS.md` 至少记录总体状态、当前阶段、当前步骤、已完成、下一步、阻塞项和最后更新时间。
5. 每次阶段切换或出现阻塞时同步更新两个文件；任务完成后记录验证命令和最终结果。

简单问答、单次只读查询和无需修改项目的操作可以省略上述文件。
